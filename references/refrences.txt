Parkinson's Disease — Voice Biomarker Extraction
-------------------------------------------------

Every feature implemented here is derived directly from the peer-reviewed
literature that produced the two UCI datasets this system targets.

PRIMARY REFERENCES
------------------
[L07]  Little MA, McSharry PE, Roberts SJ, Costello DAE, Moroz IM.
       "Exploiting Nonlinear Recurrence and Fractal Scaling Properties for
       Voice Disorder Detection."
       BioMedical Engineering OnLine 2007, 6:23.
       → Defines RPDE, DFA, spread1, spread2, D2, PPE, and confirms the
         Praat-compatible jitter/shimmer/HNR pipeline for the UCI dataset.

[L08]  Little MA, McSharry PE, Hunter EJ, Spielman J, Ramig LO.
       "Suitability of Dysphonia Measurements for Telemonitoring of
       Parkinson's Disease."
       IEEE Trans Biomed Eng 2009, 56(4):1015-1022.
       → Original paper for the UCI Parkinson's classification dataset (195
         recordings, 31 subjects). Introduces PPE. Confirms Fo/Fhi/Flo
         extraction via MDVP-equivalent Praat cross-correlation pitch.

[T10]  Tsanas A, Little MA, McSharry PE, Ramig LO.
       "Accurate Telemonitoring of Parkinson's Disease Progression by
       Non-Invasive Speech Tests."
       IEEE Trans Biomed Eng 2010, 57(4):884-893.
       → Original paper for the UCI Telemonitoring dataset (5875 recordings,
         42 subjects). Defines SEVERITY_FEATURES schema and UPDRS targets.

[P96]  Boersma P, Weenink D. Praat: doing phonetics by computer.
       University of Amsterdam, 1996–present.
       https://www.fon.hum.uva.nl/praat/
       → Authoritative source for all jitter, shimmer, and HNR formulas.
         The Praat "Voice" manual pages explicitly map each measure to
         the MDVP (Kay Elemetrics) parameter name used in the UCI datasets.

[W17]  Wikipedia / Little (2007): Recurrence Period Density Entropy.
       https://en.wikipedia.org/wiki/Recurrence_period_density_entropy
       → Documents the canonical RPDE algorithm with phase-space embedding.

================================================

The recording should be a 3–6 second sustained /aah/ VOWEL(only these sounds are preffered and trained according to this) at a
comfortable pitch and loudness, recorded in a quiet environment.
This matches the protocol used to create both UCI datasets [L08, T10].

the refernces of the pitch max and min 

    pitch_floor   : Lower F0 search bound Hz. Default 75 Hz suits male and
                    female adult voices [P96].
    pitch_ceiling : Upper F0 search bound Hz. Default 500 Hz [P96].

refernces for audio preprocessing to mono

        Mono conversion is required because Praat's PointProcess operates
        on a single channel. Peak normalisation ensures shimmer calculations
        (which measure relative amplitude change) are independent of
        recording level, matching the MDVP protocol [L08].

refernces of pitch features extraction 

      METHOD  [L08, P96]
        ──────
        F0 is estimated using Praat's cross-correlation (cc) pitch tracker,
        which is the closest open-source equivalent to the MDVP
        autocorrelation-based pitch algorithm used to produce the UCI dataset.

        MDVP:Fo(Hz)  — mean F0 over voiced frames
        MDVP:Fhi(Hz) — maximum F0 (parabolic interpolation)
        MDVP:Flo(Hz) — minimum F0 (parabolic interpolation)




extraction of harmonics features(noise features)
       HNR and NHR — ratio of periodic (harmonic) to aperiodic (noise) energy.

        FORMULA  [L07, P96 Voice§4]
        ───────
        HNR (dB) = 10 · log10( harmonic_power / noise_power )

        Computed via Praat's cross-correlation harmonicity, which matches the
        MDVP GNE-based HNR computation [L07].

        NHR = noise_power / harmonic_power
            = 1 / (10^(HNR/10))
            (NOT  1/(HNR+ε) — the dB → linear conversion is mandatory)

        Healthy voices: HNR > 20 dB  (NHR ≈ 0.01)
        PD voices:      HNR < 15 dB  (NHR > 0.03) [L07]
